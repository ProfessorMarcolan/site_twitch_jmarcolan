Nesse tutorial você irá aprender a utilizar a ferramenta de log do git. Inicialmente você criou um repositório de trabalho do git utilizando o TortoiseGit . Em seguida aprendeu a utilizar a ferramenta de commit do git . Após utilizar por um certo tempo a ferramenta de commit do git você deve ter criado um certo histórico. Para poder visualizar esse histórico você vai ter que utilizar a ferramenta de log do git.

Para acompanhar esse tutorial você já deve ter instalado o TortoiseGit. Caso você ainda não tenha instalado o TortoiseGit veja o tutorial para instalar o TortoiseGit .

O comando de log do git

Por padrão o comando log do git mostra o histórico de commits feitos no repositório em ordem cronológica regressiva (do ultimo para o primeiro). Com esse comando o usuário pode ver todas as informações de cada registro de estado feito (commit) como: linhas adicionadas, linhas deletadas, a data e a mensagem do commit. Além disso, o usuário pode buscar o histórico de versionamento a partir de um critério. Por exemplo, pode-se buscar os commits feitos por um determinado autor, por uma determinado período de tempo. Ainda com esse comando pode-se verificar a diferença nos documentos entre duas versões distintas.

Usando o commando log do git para ver o histórico de versionamento como TortoiseGit

Para acessar a ferramenta log do git com o TurtoiseGit basta clicar com o botão direito na pasta do diretório de trabalho e no menu de contexto ir com o mouse na opção TortoiseGit e depois na opção Log Show conforme mostrado na figura 1:

Ao clicar na opção show log vai aparecer a interface gráfica do TurtoiseGit da ferramenta de log do git. Essa interface pode ser visualizada na figura 2. A função de cada widget da interface gráfica é apresentado em seguida

  • Widgets no topo da interface gráfica. O primeiro widget de texto (que esta escrito “master”) serve para especificar qual o ramo dos registro que serão selecionados na pesquisa dos widgtes ao seu lado.Os widgets de data “to” e “from” servem para selecionar qual intervalo de data serão mostrado nas linhas da tabela a baixo. E o ultimo campo de texto pode-se inserir um termo de pesquisa.
  • Tabela com as colunas “graph”, “actions”, “Message”, “Author”, “date”.Essa tabela serve para navegar entre os registros selecionados na pesquisa dos widgets a cima. Cada linha dessa tabela representa um registro do estado dos arquivos- um commit.
  • Campo de texto entre as tabelas. Esse campo de texto serve para vizualizar o texto informado no versionamento selecionada na tabela a cima
  • Tabela com as colunas “path”, “extension”,”status”, “lines added”, “lines removed”. Essa tabela serve para visualizar os arquivos que foram modificados no commit selecionado na tabela em cima.
