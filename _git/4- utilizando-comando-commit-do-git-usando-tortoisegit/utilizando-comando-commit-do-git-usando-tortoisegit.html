Nesse tutorial você irá aprender a utilizar a ferramenta de commit do git. No tutorial passado você criou um repositório de trabalho do git utilizando o TortoiseGit . Agora você já possui esse diretório de trabalho no qual é possível utilizar o comando de versionamento do git.

Para acompanhar esse tutorial você deve ter instalado o TortoiseGit. Caso você ainda não tenha instalado o TortoiseGit veja o tutorial para instalar o TortoiseGit .

Como funciona a ferramenta de commit do git

A ferramenta de commit do git é a ferramenta responsável por gravar o estado atual dos arquivos. Para o Git inicialmente todos os arquivos no diretório de trabalho não estão sendo monitorados (não versionados). A partir do momento que o arquivo é versionado (gravado o estado) uma unica vez ele começa a ser monitorado pelo Git.

O git considera que os arquivos que estão sendo monitorados estão entre dois estados: inalterados ou modificados. Esses dois estados servem para o Git decidir se o arquivo precisa possuir um novo registro de estado ou não. Caso o arquivo não tenha sido alterado comparado ao ultimo registro de estado dele o sistema o ignora. Caso o arquivo tenha alguma alteração o sistema permite a aquisição de um novo registro desse documento. Dos arquivos habilitados para um novo versionamento pelo Git o usuário pode escolher quais desses arquivos ele quer gravar o estado. Assim, os arquivos alterados voltam ao estado de inalterados. Conforme o uso dessa ferramenta o usuário vai criando uma série de relatórios de estados dos arquivos. Essa série de relatórios posteriormente podem ser visualizados.

Utilizando a ferramenta de commit do git com o TortoiseGit

Para testar essa ferramenta crie um documento de texto (txt) dentro da pasta de trabalho, criado no tutorial passado, com o nome “primeiroVersionamento.txt” e escreva dentro dele a frase “Primeiro versionamento” e salve. O arquivo deve ficar como o da Figura 4.

Figura 4. Arquivo de versionamento

Após salvar o arquivo você irá criar o primeiro versionamento desse arquivo. Para realizar isso, clique novamente com o botão direito dentro da pasta de trabalho do git e dentro do menu de contexto clique no comando “Git commit -> master” (como mostrado na figura 5).

Figura 5. Arquivo de versionamento

Interface gráfica do TortoiseGit da ferramente Commit do Git

Após clicar no “Git commit -> master” vai abrir a interface gráfica do TortoiseGit relacionada a ferramenta de commit (conforme figura 6). As funcionalidades de cada widget dessa interface são descritos a seguir).

Figura 6 Interface gráfica do TortoiseGit relacionada a ferramenta de commit do git

Na caixa que está escrito “messagem”:

  • Escreva uma mensagem no campo de texto. Esse campo serve para o autor comentar o que foi alterado nos documentos desse versionamento
  • Marque a opção “set author date”. Esse botão serve para informar quando o autor fez o registro do versionamento arquivos
  • Marque a opção “set author”. Esse botão serve para informar quem foi o autor do versionamento dos arquivos

Na caixa que está escrito “changes made” :

  • Marque a caixa dos arquivos que você quer selecionar para o versionamento na tabela que contem os arquivos. A tabela tem as colunas “path”, “extension”, “status”, “lines add”, “lines removed”. A coluna “path” informa o caminho do arquivo, a coluna “extension” informa a extensão do documento, a coluna “Status” informa o status do versionamento de arquivo- se
    for unknown é porque não foi ainda versionado, A coluna “lines add” diz respeito a quantas linhas foram adicionados no documento e a coluna “lines subtraction” quantas linhas foram retiras
  • Marque a opção “show Unversioned Files” . Essa caixa de opção serve para mostrar na tabela os arquivos que ainda não foram versionado, não estão ainda monitorados

Por fim clique no botão commit . Esse botão serve executar o comando de versionamento – ira abrir uma interface gráfica como mostrado na figura 7;

Figura 7. Funcionou o commit do git

Pronto! Seu primeiro versionamento foi feito com sucesso. Aperte no botão “close” para fechar essa janela.

Próximo post: como usar ramos e mesclar código

Durante esse tutorial você deve ter visto varias vezes a palavra “master”. Ela esta relacionada ao ramo em que estamos adicionando o histórico de estado dos arquivos. Com o git, o histórico não precisa ser feito necessariamente em uma unica linha temporal. Com a ferramenta de ramos do git é possível criar diversas linhas temporais de versionamento. O git posteriormente consegue mesclar o registro de estados dos documentos dessas várias linhas temporais.
