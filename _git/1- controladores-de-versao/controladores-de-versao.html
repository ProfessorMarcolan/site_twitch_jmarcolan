Imagine ter o histórico de todos os seus códigos. Imagine editar seu código sem ter o medo dele parar de funcionar na próxima atualização do código. Tudo isso é possível utilizando um controlador de versão.

Controladores de versão

Nesse post iremos conversar sobre controladores de versão. Um sistema controlador de versão é um software que tem a finalidade de administrar as diferente versões de código fonte e documentação dos projetos. Esses sistemas são largamente utilizados no desenvolvimento de softwares pois, permitem com facilidade registar o histórico do desenvolvimento do projeto e gerir as diferentes versões (eg. retornar o código fonte a uma versão anterior caso não seja estável). Outro motivo é porque eles permitem várias pessoas trabalharem num mesmo projeto. Os sistemas com soluções livres mais comuns são: Git , SVN, CVS, Mercurial.

O comando Commit

A função mais básica do sistema de controle de versão é a “commit”. Essa função envia as ultimas alterações para um repositório que o armazena de tal forma que o sistema consegue manter um histórico de todo o desenvolvimento do projeto. Isso permite resgatar, caso necessário, qualquer versão mais antiga do código. Assim sendo, é recomendado usar a função “commit” a cada alteração que o software estiver minimante estável. Pela função “commit” também é possível acrescentar comentários ao envio de alterações, o que facilita a análise do histórico.

OS comandos Branch e Merge

Por padrão os sistemas controladores de versão guardam o histórico de alteração do projeto de forma sequencial em um único ramo chamado geralmente de “Master”. Porém, alguns SCV conseguem criar vários fluxos paralelos ao principal e depois, caso necessário, mesclar os de interesse (Merge). A função responsável por criar ramificações no histórico principal do projeto é a “Branch”. As ramificações são convenientes para o teste de novas idéias porque permitem criar uma distinção entre as linhas de desenvolvimento dos pensamentos. Essa desconexão entre os progressos permite que os ramos possam ser trabalhados de forma paralela por diversas pessoas. Ao final do desenvolvimento das diversas linhas de pensamento elas podem ser mescladas pela função “Merge”.

O Git

Essa filosofia de desenvolvimento é chamada de filosofia Otimista de mesclagem e é geralmente padrão na maioria dos sistemas abertos como o Git. O Git é um software livre, distribuído sob os termos da versão 2 da GNU General Public License. Sua manutenção é atualmente supervisionada por Junio Hamano e foi desenvolvida inicialmente por Linus no inicio de 2005.
