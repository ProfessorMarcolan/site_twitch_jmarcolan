O paradigma da Programação orientada ao objeto é uma das formas mais efetivas de escrever softwares[4]. Se você utilizar de forma correta você consegue evitar que seus códigos cheirem mal . Várias linguagens de programação permitem utilizar a programação orientada ao objeto. Aqui você pode encontrar tutoriais de java sobre a programação orientada ao objeto . Agora para o Python essa ideia é ainda mais central, porque como veremos mais para frente “tudo” é considerado um objeto. Então, nessa linha de tutoriais iremos entender como funciona esse paradigma de programação e como criar nossos próprios objetos.

Paradigma de orientação ao objeto em Python

No paradigma da orientação ao objeto você escreve programas a partir da criação de classes . Podemos definir as classes como uma representação, um reprodução ou um modelo. As classes representam situações/coisas. Quando você escreve uma classe , você define um comportamento geral que toda uma categoria de objetos pode ter. Quando você constrói um objeto utilizando a classe você automaticamente equipa esse objeto com todos os comportamentos gerais. As classe tem dados e métodos. Para cada objeto você pode setar seus traços únicos [4]. Uma analogia didática seria dizer que as classes são como as receitas de um bolo, os objeto são o próprio bolo, os atributos são os ingredientes e os métodos são as ações para o preparo desse bolo.

O processo de instanciação é o processo de construir um objeto a partir de uma classe . Você trabalha com instâncias da classe (os objetos) . Criando uma Classe em Python

Definir uma classe em python é bem simples. Uma classe em Python começa com a palavra reservada class e em seguida pelo nome da classe seguido pelos parenteses ( () ) e dos dois pontos (:) . A partir da definição do nome da classe tudo dentro da identação (quatro espaços) após a prototipação pertence a classe . E a primeira linha de código sem a identação está fora da classe. Você pode uma classe simples no código 1.

#código 1 class PrimeiraClasse(): pass # A palavra reservada pass não fas nada.

Essa classe ainda não possui nenhum método ou dado. A palavra reservada pass não faz nada. Ela é silenciosamente ignorada e geralmente é adicionada quando o programador estiver trabalhando em um código novo ou então quando o programador quer permitir abstrações como veremos mais para frente.

Instanciando uma classe em Python para criar um objeto

Pelo código 1 escrevemos nossa primeira classe. Agora no código 2 iremos instanciar essa classe para criar um novo objeto dessa classe. Para isso iremos criar uma variável p1 e atribuir ela a classe “PrimeiraClasse”.

#código 1 class PrimeiraClasse(): pass # A palavra reservada pass não fas nada. #código 2 Instanciando um novo objeto p1 = PrimeiraClasse() print(type(p1)) #

A linha 3 estamos instanciando um novo objeto. Como pode-se observar instanciar classes em Python é bem direto. Basta pensa-la como uma função que retorna um novo objeto. Na linha 4 estamos imprimindo o tipo da variável p1. Como observaremos a saída nos dirá que é uma variável do tipo “PrimeiraClasse”. .

Métodos construtores e destrutores em Python

Em outras linguagens que possuem o paradigma da orientação ao objeto (e.g. c++) os objetos possuem dois métodos especiais: o método que é invocado pelo sistema quando se cria um objeto e o método que o sistema chama quando se deleta um objeto. Isso é feito porque diferentes objetos necessitam de diferentes passos para serem construídos e destruídos. Em uma analogia didática é pensar como pensar na diferença entre os passos necessários para construir uma casa em relação a um carro. E eles também possuem passos bem diferentes para serem destruídos.

O método que é chamado para criar o objeto é o construtor e o método que é invocado quando é necessário exclui-lo é o destrutor. Em outras linguagens isso é um tópico super importante pois os programas podem possuir vazamento de memória caso esses métodos sejam implementados de forma inapropriada. O Python retira um pedaço desse trabalho/responsabilidade do programador através da ideia do método inicializador ( __init__ ).

O método inicializador __init__

O Python te permite instruir a nova criação de um objeto com um método inicializador chamado __init__ . O método __init__ não é o construtor. O método construtor chama o método ___init__ caso haja um implementado. Em Python o objeto já esta construído quando ele chama o método inicializador __init__() . O inicializador apenas configura o já criado objeto antes desse objeto retornar a quem solicitou esse novo objeto. É por isso que é possível construir objetos em Python sem precisar implementar esse método em seu código. No código 3 iremos modificar a classe “PrimeiraClasse” para constar com o método __init__ . E novamente iremos instanciar um objeto usando essa classe.

#código 3 - inicializando uma classe class PrimeiraClasse(): def __init__(self): print("inicializou nosso objeto") p1 =PrimeiraClasse() #inicializou nosso objeto

Agora ao rodar o código 3 percebemos que na tela estará impresso a mensagem “inicializou nosso objeto”. Isso acontece porque quando instanciamos um novo objeto o Python invoca o construtor do objeto que por sua vez chama o nosso método inicializador. E no nosso método inicializador possui uma função de escrever na tela ( print ) que tem como entrada um argumento string . Observando bem o nosso método __init__() também possui um argumento de entrada o self

O argumento self dentro da classe em python

Quando você definir os métodos da sua classe você explicitamente deve listar o argumento “self” como o primeiro argumento de cada método, incluindo o método __init__. E como veremos mais para frente isso também deve ser feito quando você chamar um método de uma superclasse (classe pai, classe base). Porém, quando você chamar os métodos de fora da classe você não precisa adicionar o parâmetro “self”. Você pode pular totalmente esse argumento. O Python vai adicionar de forma automática a referencia da instância para você.[3] Isso vai parecer confuso de inicio mas isso não é feito de forma inconsistente. É feito de forma consistente porque o Python confia na distinção entre métodos bound e unbound . [3]

Criando um método para o objeto

Iremos agora modificar nossa classe “PrimeiraClasse” para ela ter um método implementado por nós e vamos acessar esse método de fora da classe. Para isso segue o código 4.

#código 4 - criando um meotodo dentro da classes #invocando esse metodo de fora da classe. class PrimeiraClasse(): def __init__(self): print("inicializou nosso objeto") def primeiroMetodo(self): print("primeiro método implementado") p1 =PrimeiraClasse() #inicializou nosso objeto p1.primeiroMetodo() #primeiro método implementado

Observando o código 4 pode-se perceber que dentro da classe o método “primeiroMetodo” possui como argumento o parâmetro “self” porém quando invocamos ela fora da classe não passamos nenhum argumento. Para acessar o método da instância utilizamos o operador ponto (.) . O operador ponto ponto é o operador que é usado para acessar os métodos e os dados da instância. Quando invocamos o método “priemiroMetodo” percebemos que no prompt de comando esta escrita a frase “primeiro método implementado ” .
